% Problem Statement Analysis
\section{Problem Statement Analysis}

\subsection{Background and Context}

The safety of electrical installations critically depends on reliable Miniature Circuit Breakers (MCBs). The IEC 60898-1:2015 standard mandates rigorous short-circuit breaking capacity tests to ensure MCBs perform correctly under severe fault conditions involving currents up to 10,000A.

\subsection{Existing Challenges}

Current manual and semi-automated testing methods face several critical issues:

\begin{enumerate}
    \item \textbf{Safety Risks}: Personnel exposure to high-energy fault conditions (up to 10,000A)
    \item \textbf{Imprecise Control}: Manual R (resistive) and XL (inductive) circuit configurations
    \item \textbf{Time Inefficiency}: Increased test times due to manual processes
    \item \textbf{Repeatability Issues}: Human error affecting test accuracy and consistency
    \item \textbf{Limited Real-time Analysis}: Lack of live waveform visualization and data analysis
    \item \textbf{Communication Gaps}: Poor integration between hardware controllers and analysis software
\end{enumerate}

\subsection{Expected Solution Requirements}

The automated machine must deliver automated control of test currents, voltages, and circuit impedance with support for multiple MCB types (0.5A-63A), real-time power factor control, high-speed data acquisition, comprehensive safety systems, and full IEC 60898-1:2015 compliance.

\section{Our Journey: Day-by-Day Evolution}

\subsection{Day 1: Foundation - Automation Logic \& Control System}

\textbf{Initial Approach: ``TripIQ'' - The Smart Testing Brain}

Our first day focused on building the core automation infrastructure that would serve as the foundation for MCB testing.

\textbf{Key Developments:}
\begin{itemize}
    \item \textbf{Custom GUI Development}: Built ``TripIQ'' software using Python (PyQt5) as the central command center
    \item \textbf{Hardware Communication}: Established USB-based communication with microcontroller systems
    \item \textbf{Relay Matrix Implementation}: Designed automatic switching between multiple MCBs (MCB 1-4)
    \item \textbf{Data Acquisition System}: Integrated current sensors with Pandas/Matplotlib for real-time analysis
\end{itemize}

\textbf{Technical Stack (Day 1):} Python 3.10+, PyQt5, NumPy/Pandas, Matplotlib, PyUSB/Serial communication, microcontroller-based relay matrix.

\textbf{Achievements:} Automated MCB switching, real-time data capture, operator-free testing loop, scalable software architecture.

\subsection{Day 2: The Pivot - AC Power Solution \& Real-time Control}

\textbf{The Challenge}: During evaluation, judges required a shift from logic-focused automation to comprehensive \textbf{AC Power Control} with dynamic power factor management.

\textbf{New Requirements:} Real-time AC current control, software-controlled power factor adjustment (0.5 to 0.9), live waveform visualization with phase relationships, and precise impedance matching.

\textbf{Our Response: Advanced AC Impedance Control System}

We completely redesigned our approach to include:

\begin{enumerate}
    \item \textbf{Software-Defined Power Factor Control}:
    \begin{itemize}
        \item User inputs target Voltage, Current, and Power Factor via GUI
        \item Automatic calculation of required Resistance (R) and Inductive Reactance (XL)
        \item Real-time impedance matching using the formula: $PF = \frac{R}{\sqrt{R^2 + X_L^2}}$
    \end{itemize}
    
    \item \textbf{Digitally Controlled Variable Impedance Bank}:
    \begin{itemize}
        \item Bank of precision resistors and inductors
        \item Heavy-duty contactors for automatic R-L combination switching
        \item Microcontroller-based selection of optimal impedance values
    \end{itemize}
    
    \item \textbf{Real-time Waveform Analysis}:
    \begin{itemize}
        \item Live voltage and current waveform display
        \item Phase difference visualization with angle measurements
        \item DC offset removal and cycle looping for continuous waveforms
    \end{itemize}
\end{enumerate}

\textbf{Enhanced Technical Architecture:} PyQt5 frontend, Python TCP/UDP backend, ESP32 WiFi controller, Matplotlib visualization, TCP protocol (10.91.136.24:8888), NumPy signal processing.

\subsection{Day 3: Integration \& Real-world Testing}

\textbf{Focus}: System integration, real hardware testing, and performance optimization.

\section{Technical Implementation Details}

\subsection{System Architecture}

Three-tier architecture: Frontend (PyQt5 GUI with 9 tests, configurations, waveform visualizer) → Backend (Communication module, signal processing, R-L control algorithm) → Hardware (ESP32 controller, voltage sensors, R-L matrix switching, MCB test station) connected via TCP/WiFi (Port 8888).

\subsection{ESP32 Integration \& Communication}

ESP32 controller (10.91.136.24:8888) provides real voltage measurements with 100 samples/cycle, 10-second intervals, high-precision ADC with timestamps. Communication protocol: Commands "current_value,power_factor\textbackslash n", Responses "voltage,timestamp@".

\subsection{Variable R-L Configuration System}

The system provides precise control over resistance and inductance values through a sophisticated relay-controlled impedance bank:

\begin{equation}
    \text{Power Factor} = \cos(\phi) = \frac{R}{\sqrt{R^2 + X_L^2}}
\end{equation}

where $X_L = 2\pi f L$ is the inductive reactance at 50Hz.

\subsection{Hardware Components and Circuit Design}

The circuit diagram shows our innovative R-L matrix switching system with the following key components:

\textbf{Power Supply:}
\begin{itemize}
    \item 50Hz AC Source: Primary power supply for MCB testing
    \item Voltage Range: 230V AC (standard domestic supply)
    \item Frequency: 50Hz (Indian power grid standard)
\end{itemize}

\textbf{Inductance Bank:}
\begin{itemize}
    \item 30mH Inductor: High-current air-core inductor for low impedance
    \item 500mH Inductor: Medium inductance for moderate power factor control
    \item 10mH Inductor: Fine-tuning inductance for precise power factor adjustment
    \item 40mH Inductor: Additional inductance value for extended range
\end{itemize}

\textbf{Resistance Bank:}
\begin{itemize}
    \item 1.4 Ohm Resistor: Low resistance for high current applications
    \item 47 Ohm Resistor: Medium resistance for balanced impedance
    \item 30 Ohm Resistor: High resistance for low current testing
    \item 7.2 Ohm Resistor: Intermediate resistance for fine control
\end{itemize}

\textbf{Relay Control Matrix:}
\begin{itemize}
    \item Relay 1: Controls 30mH inductor and 1.4 Ohm resistor path
    \item Relay 2: Switches 500mH and 40mH inductors with 800mH combination
    \item Relay 3: Controls 47 Ohm resistor in series configuration
    \item Relay 4: Manages 30 Ohm resistor for high impedance scenarios
    \item Relay 5: Controls 7.2 Ohm resistor for intermediate impedance
    \item Relay 6: Master control for entire circuit activation
\end{itemize}

\textbf{MCB Test Station:}
\begin{itemize}
    \item Universal MCB Slot: Accommodates various MCB types (6A to 63A)
    \item Quick-connect terminals for easy MCB insertion/removal
    \item Safety interlocks to prevent accidental energization
    \item Arc chute protection for safe fault current interruption
\end{itemize}

\textbf{Circuit Operation Principle:}

The R-L matrix operates on the principle of selective impedance switching:

\begin{enumerate}
    \item \textbf{Software Calculation}: Target power factor input triggers impedance calculation
    \item \textbf{Relay Selection}: Algorithm determines optimal R-L combination
    \item \textbf{Sequential Switching}: Relays activate in predetermined sequence to avoid arcing
    \item \textbf{Impedance Verification}: Current and voltage sensors confirm actual impedance
    \item \textbf{Fine Tuning}: Micro-adjustments through parallel/series combinations
\end{enumerate}

\textbf{Safety Features Implemented:}

\begin{itemize}
    \item \textbf{Arc Suppression}: RC snubber circuits across relay contacts
    \item \textbf{Overcurrent Protection}: Fast-acting circuit breakers (response time $<$5ms)
    \item \textbf{Isolation Barriers}: Optical isolators between control and power circuits
    \item \textbf{Emergency Stop}: Hardware-based immediate disconnection
    \item \textbf{Thermal Monitoring}: Temperature sensors on high-power components
    \item \textbf{Ground Fault Detection}: Residual current monitoring
\end{itemize}
